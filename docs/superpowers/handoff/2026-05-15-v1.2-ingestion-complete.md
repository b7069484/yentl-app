## Known gaps / follow-ups

These were noticed during the visual sweep but are NOT blockers for v1.2 shipping:

1. **No auto-redirect to overview after bulk-ingest completes.** Text-doc pane stays on the textarea even after the pipeline produces claims. User has to click "Overview" manually. Fix: in `text-ingest-pane.tsx` (and the other ingest panes), navigate to `/session?view=overview` after `bulkIngest` resolves. ~10 min fix.

2. **Ingest pane button styling inconsistency.** Process / Fetch buttons use `bg-primary` Tailwind class which is rendering as near-black instead of teal. Compare to the Mic pre-record button which uses `bg-teal` directly and looks right. Audit and standardize.

3. **"Listening for voicesâ€¦" speaker rail copy renders even when no session has started.** Pre-existing (in `speaker-rail.tsx`), not introduced by this sprint. The copy is misleading in the picker-stage and mic-prerecord stage.

4. **`isRecording` semantics for non-mic sources** â€” set true by `startSession()` but never actually "recording" for text/audio/youtube/media. Currently appears to settle at false after bulkIngest completes (which is fine). The header pill shows "Paused" momentarily. Consider extending the pill state machine to "Processing" for non-mic sources.

5. **Sentence splitting** â€” observed during text-doc test: 2 sentences ("â€¦in the 1960s. We have createdâ€¦") came through as 1 transcript segment. Either `Intl.Segmenter` is treating `1960s.` as a non-terminal, or the regex fallback fired incorrectly. Worth a dedicated test fixture.

6. **Audio file end-to-end test deferred** â€” `BLOB_READ_WRITE_TOKEN` not present in `.env.local`. UI verified; full E2E pending `vercel env pull`. No code issue.

7. **YouTube timedtext access tightening** â€” YouTube has reportedly been restricting timedtext access. Several test videos returned `NO_CAPTIONS` even though they have captions on the player. Worth a real-world reliability investigation before v1.2 ships to users.
