## Status Values

| Status | Meaning |
|---|---|
| `candidate` | Proposed from plans/audits; needs scoping before a build loop can act. |
| `ready_for_build` | Narrow, accepted, and verifiable. |
| `in_progress` | A build loop selected it this run. |
| `built_pending_verify` | Code changed and local checks ran; later audit/watchdog must verify. |
| `verified_done` | Later pass confirmed the slice works and did not regress. |
| `reopened` | The slice failed later verification or regressed. |
| `blocked_needs_context` | Needs human/product decision. |
| `blocked_needs_verification` | Work may exist but proof could not run. |
| `deferred` | Intentionally postponed with rationale. |
| `escalated` | Repeatedly blocked or high-risk. |
