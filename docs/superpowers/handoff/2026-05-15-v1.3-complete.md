## Feature-by-feature

### T10 — Karaoke watch view + utterance merging
- Right panel is now the live transcript itself, every line visible
- State per line: `past` (muted ink-2), `current` (teal ring + bg-teal-soft + ▶, last line whose start ≤ currentTime), `future` (faded ink-4)
- Auto-scroll on current-line change via `scrollIntoView({ block: "center", behavior: "smooth" })`
- Click any line to seek
- Inline claim/marker annotations under their anchor segment, suppressed for future lines to avoid spoilers
- `lib/client/utterance-merge.ts` — merges 3-second SRT fragments into 80–360 char sentence-bounded utterances. Fine-grained segments still go into the store for karaoke display; merged utterances go to `/api/extract-claims` so the extractor has real sentences to work with. Drops `[Music]`/`[Applause]` cues.

### T11 — Sessions library (local-first IndexedDB)
- `lib/client/session-storage.ts` — IDB CRUD using the native API (no external runtime dep). Schema: `yenta` DB, `sessions` store keyed on ulid id, indexed on `saved_at`.
- New route: `/sessions` — list view with restore/rename/delete per card.
- Save button in session header opens a dialog with name pre-filled from title; survives page reload, lost on cache clear.
- `restoreSession(session)` store action overwrites all relevant fields and resets volatile state.
- "Library" link in the session shell header next to the brand; "Sessions library" CTA on the landing page.
- Dev dep: `fake-indexeddb` for testing.

### T12 — Per-speaker verdict in synthesis
- `/api/synthesize` Zod schema extended with `per_speaker_verdicts: SpeakerVerdict[]` (optional). Each verdict has `speaker_id`, `label`, `factual_grade`, `faith_grade`, `one_liner`.
- System prompt gained a grading rubric: rules for when to use `mostly_factual` / `mixed` / `mostly_inaccurate` / `insufficient` and the equivalent faith grades; explicit "be conservative, require a pattern" instructions.
- `SynthesisCard` renders the verdicts as side-by-side cards (desktop) / stacked (mobile) under the synthesis paragraph: speaker name + colored dot, two grade chips, italic one-liner.

### T13 — Audio + media URL E2E without Vercel Blob
- Removed `@vercel/blob` dep + `/api/upload-audio` route
- `/api/transcribe-batch` now accepts multipart/form-data (with the file directly) AND the existing JSON body (back-compat for media URL)
- `lib/server/deepgram-batch.ts` added `transcribeFile(buffer, mime)` alongside `transcribeUrl(url)`
- Audio pane: `URL.createObjectURL(file)` is set as `source.blob_url` so the Watch view's audio adapter can play back the local file. Cleanup revokes the object URL.

### T14 — Diarization tuning + manual speaker re-attribute
- Deepgram params: `numerals: true` added to both live WS and prerecorded paths. `diarize_version` and `vad_events` checked but not present in v5 SDK types — skipped.
- Store actions: `reassignUtterance(transcriptIndex, newSpeakerId)` updates the segment AND cascades to claims (matching `utterance_start`) and markers (within the segment's time range). `addNewSpeaker()` returns max+1 and registers.
- `components/session/reassign-speaker-menu.tsx` — Radix DropdownMenu popover with all known speakers + "Add new speaker" option.
- Wired into the Watch transcript (each past/current line gets the menu) and the Transcript tab.

## Known issues for next session

1. **YouTube in production (Vercel deploy)** — `yt-dlp` is on PATH locally but won't be on Vercel's serverless runtime. Two options:
   - Bundle yt-dlp static binary via Vercel custom build setup
   - Run YouTube ingest in Vercel Sandbox or a separate worker process
   Defer until you actually try to deploy.

2. **Audio file upload over 100MB on Vercel** — server-mediated approach holds the full file in memory inside the function. Fine for ~50MB; risky for the 500MB cap. If you start hitting timeouts, swap back to Vercel Blob (now that the user knows what `vercel link && vercel env pull` does) or use a streaming upload pattern.

3. **Caption extraction quality on caption-only YouTube videos.** Even with the utterance-merge fix, fully accurate claim extraction depends on the captions being clean. If yt-dlp's auto-captions are noisy (lots of `[♪♪♪]` or partial words), the extractor will be sparse. The user should expect best quality from videos with manually-curated captions (TED talks, news transcripts, etc.).

4. **Reassign menu coverage in TranscriptView is "first index of block" only.** Mid-block reassignment isn't possible without splitting the speaker block. Future enhancement: split a block at the click point.

5. **Watch tab visible for source.kind ≠ mic & text_doc** — but it shows "No media to watch" for those two. That message is OK but a missed opportunity to redirect to /overview directly.

6. **Saved session restore doesn't replay synthesis** — `restoreSession` clears `synthesis` to null. If the user restores a saved session and wants the AI commentary back, they'd have to re-trigger synthesis manually. Consider persisting synthesis alongside the rest.
