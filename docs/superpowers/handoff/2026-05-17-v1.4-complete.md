## Headline

Six v1.3-handoff candidates landed and one critical Vercel-bundling fix on top: YouTube ingest is now production-deployable, the synthesis paragraph + per-speaker verdicts survive save/restore, large audio uploads stream instead of buffering, deep-link-to-watch-with-mic redirects cleanly, mid-block utterance reassignment is possible, Watch annotations link to L3 detail, and the player frame carries a speaker-colored ring that transitions on speaker change.

| | Before v1.4 | After v1.4 |
|---|---|---|
| Save → restore → synthesis | Lost (set to null on restore) | Preserved (text + headlines + per-speaker verdicts) |
| YouTube on Vercel | `YT_DLP_MISSING` (yt-dlp not on serverless PATH) | `youtube-dl-exec` bundled binary + `outputFileTracingIncludes` glob covering pnpm symlink layout |

## Task-by-task

### A — Synthesis persistence in saved sessions
- `lib/types.ts`: added `synthesis?: PersistedSynthesis` to `Session` and the new `PersistedSynthesis` shape (`{ text, headlines, per_speaker_verdicts?, at }` — no `state` discriminant).
- `lib/client/session-store.ts`: `toSession()` includes synthesis only for `"fresh"` and `"refreshing"` states. `restoreSession()` rebuilds it as `"fresh"` (refreshing downgraded — no live job to restore).
- `SpeakerVerdict` deduped — single source of truth in `lib/types.ts`, re-exported from `session-store.ts` for back-compat. The old comment claiming a "circular dependency" was inaccurate.
- 12 unit tests in `tests/synthesis-persistence.test.ts` cover every state branch.

### B — Watch tab redirect when source has no media
- `app/session/page.tsx`: when `view === "watch"` and `source.kind` is not in `PLAYABLE_SOURCE_KINDS`, `useRouter().replace("/session?view=overview")` fires in a `useEffect` after mount.
- `components/session/watch-view.tsx`: the entire `if (source.kind === "mic" || source.kind === "text_doc")` no-media block (lines 353–371) was removed — it's now unreachable.
- `components/session/session-shell.tsx`: `PLAYABLE_SOURCE_KINDS` exported so page.tsx can import it.
- 6 redirect tests cover all five source kinds + the non-watch view path.

### C — Streaming audio upload for files >50MB
- `app/api/transcribe-batch/route.ts`: new `STREAM_THRESHOLD_BYTES = 50 MB`. Files at or below stay on the existing Buffer path; files above are converted via `Readable.fromWeb(file.stream())` and passed directly to a new `transcribeStream()` helper.
- `lib/server/deepgram-batch.ts`: `transcribeStream(stream: Readable, mime: string)` added. Deepgram v5 SDK accepts a Node `Readable` as `Uploadable.WithMetadata.data`, so no tempfile is needed.
- `bde54de` follow-up: route now falls back to `audio/mpeg` when `file.type` is empty (some browsers omit it).
- 11 new tests verify the threshold branch + error handling.

### D — yt-dlp production packaging
- `package.json`: added `youtube-dl-exec@^3.1.7` + `pnpm.onlyBuiltDependencies` whitelist (lets postinstall download the binary under pnpm 10).
- `lib/server/yt-dlp-binary.ts`: `getYtDlpBinaryPath()` resolver — env override → bundled binary via `createRequire("youtube-dl-exec/src/constants")` → PATH fallback.
- `lib/server/yt-dlp-runner.ts`: re-exports `getYtDlpBinaryPath`.
- `lib/server/youtube-captions.ts:251`: `spawn("yt-dlp", args)` → `spawn(getYtDlpBinaryPath(), args)`.
- `next.config.ts`: `outputFileTracingIncludes` with both the pnpm real-path glob AND the symlinked glob — `@vercel/nft` resolves symlinks before matching, so the real-path glob is the load-bearing one.
- `vercel.json`: `functions.maxDuration: 60` for the YouTube route.
- 5 unit tests + verified `.next/server/app/api/youtube-ingest/route.js.nft.json` includes the binary.

**Vercel deploy steps (NOT executed in this sprint):**
1. `pnpm install` runs `youtube-dl-exec` postinstall on Vercel build → downloads ~3.2MB Linux x64 binary
2. Next.js bundles it into the function via `outputFileTracingIncludes`
3. Binary is `chmod 755` from the postinstall, Vercel preserves it
4. If EACCES errors appear: add `"vercel-build": "next build && chmod +x node_modules/.pnpm/youtube-dl-exec@*/node_modules/youtube-dl-exec/bin/yt-dlp"` to package.json scripts
5. Optional `YT_DLP_PATH` env var to override the resolved path

### E — Reassign menu split-and-reassign mid-block
- `lib/client/session-store.ts`: new `splitSegmentAt(index, splitTime, newSpeakerId)` action. Splits the transcript segment at the nearest word boundary to the time fraction. Cascades claims with `utterance_start ≥ splitTime ∧ ≤ original_end` to the new speaker; cascades markers FULLY inside the post-split range; markers straddling the split are unchanged. Idempotent speaker registration. No-op on out-of-bounds.
- `components/session/reassign-speaker-menu.tsx`: new `<DropdownMenu.Sub>` "Split & reassign…" with two-step flow — pick a word chip (all-but-last words shown), then pick the speaker for the post-split portion. Back button to return to word picker. Disabled when segment has only one word.
- `computeSplitTime(seg, wordIndex, totalWords)` exported for unit testing.
- 16 store tests + 10 UI tests (including the Back-button path).

### F — L3 click affordance in Watch annotations
- `components/session/watch-view.tsx` `AnnotationRow`: refactored from single `<button>` to a `<div>` with two children:
  - `<button data-testid="annotation-chip-btn" onClick={onSeek}>` wrapping rail + chip
  - `<Link data-testid="annotation-detail-link" href="/session/detail/{claim|marker}/{id}">` wrapping italic quote + ChevronRight icon (opacity 0.4 → 1.0 on hover, transition-colors)
- Hover state scoped via `group/link` named-group modifier — isolated from the outer row hover.
- Existing `data-testid="annotation-{kind}-{id}"` preserved on the outer `<div>` so prior tests continue to work.
- 6 new tests cover chip-seeks/text-navigates/chevron-present for both claim and marker rows.

### G — Color-coded speaker-ring on player
- `components/session/watch-view.tsx`: `currentSpeakerId` derived via `useMemo` from `currentSegStart` → segment.speaker_id. Player container wrapped with `border-[3px] transition-colors duration-300` + dynamic `paletteFor(speakerId).border` class (or `border-transparent` when no current segment).
- Uses `paletteFor` imported from `TranscriptView` — a known coupling; v1.5 candidate to extract `paletteFor` + palette to `lib/speaker-palette.ts`.
- 5 new tests cover ring-applied/ring-absent/mid-session transition states.

### H — Reliability sweep

Workaround: `YT_DLP_PATH="/opt/homebrew/bin/yt-dlp" pnpm dev` (or `.env.local` entry). **Production (Vercel) is unaffected** — Vercel runs Node directly, the bundled binary path resolves normally.

## Known issues for next session (v1.5)

1. **YT_DLP_PATH dev workaround.** Static import in `lib/server/yt-dlp-binary.ts` may sidestep the Turbopack resolution issue — try replacing `_require("youtube-dl-exec/src/constants")` with `import { YOUTUBE_DL_PATH } from "youtube-dl-exec/src/constants"` (the package is CJS so this may need `import * as`). Or document `YT_DLP_PATH` in `.env.local.example`.
2. **Full-content synthesis on long-form ingests.** Current pacer runs synthesis on the trailing window. For 15-min talks the synthesis describes the closing minutes, not the whole talk. Consider triggering a final full-content synthesis on `endSession()` for non-mic sources.
3. **`paletteFor` coupling.** `components/session/watch-view.tsx` imports `paletteFor` from `TranscriptView.tsx`. Move `SPEAKER_PALETTE` + `paletteFor` to a shared module (`lib/speaker-palette.ts`).
4. **`PLAYABLE_SOURCE_KINDS` placement.** Lives in `session-shell.tsx`; page.tsx imports across the cross-layer boundary. Extract to `lib/source-kinds.ts`.
5. **Watch redirect flash.** `return null` before the `useEffect`-driven redirect causes a one-frame empty render on hard refresh / deep-link. For zero-flash, would need middleware-side redirect, but session state lives client-side. Acceptable for now.
6. **`computeSplitTime` exported for tests.** Move to `lib/client/utils.ts` so it can be unit-tested without exposing a component-internal API.
7. **Test depth — Task C streaming.** Tests verify the branch picker correctly routes large files to the stream path, but don't actually consume the stream end-to-end through the SDK in any test. Worth adding when a real failure mode surfaces.
8. **Reassign menu test workaround.** Uses `fireEvent.click` because `userEvent.click` triggers Radix's pointer-outside dismiss. Acceptable; comment explains why.
9. **Production deploy still pending user authorization.** All code is deploy-ready per the trace JSON, including yt-dlp bundling. Awaiting explicit `vercel deploy` go.
