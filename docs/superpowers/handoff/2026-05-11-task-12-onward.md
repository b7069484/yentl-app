## Critical gotchas discovered along the way

These are reasons the plan-as-written wouldn't work and what was fixed. Future sessions should keep these in mind:

1. **Deepgram SDK v5 has a totally different API from v4.** The plan's `manage.createProjectKey(...)` doesn't exist. The right call is `client.auth.v1.tokens.grant({ ttl_seconds: 600 })` which returns `{ access_token, expires_in }`. Used in `lib/server/deepgram.ts`.

2. **Deepgram WebSocket subprotocol differs by auth type.** API keys use `["token", apiKey]`. JWTs (what we mint) use `["bearer", jwt]`. The plan and the SDK README both showed `["token", key]` — this is wrong for JWTs. Fixed in commit `8a36b3b`. Confirmed against `@deepgram/sdk@5.1.0/dist/esm/CustomClient.mjs` lines 320–335.

3. **Browser `WebSocket` ignores the third `options` argument.** The SDK's `new WebSocket(url, protocols, options)` works in Node but `options.headers` is silently dropped in the browser. That's why JWT auth must go through subprotocol, not `Authorization` header.

4. **AI SDK v6 removed `generateObject`.** Replace with `generateText({ output: Output.object({ schema }) })`. The destructure becomes `const { output } = await …`. All four future API routes (extract-claims, verify-provisional, verify-confirmed, analyze-rhetoric) MUST use this pattern.

5. **AI Gateway model strings use plain `"provider/model"` with DOTS for versions.** `"anthropic/claude-opus-4.7"` — NOT `"claude-opus-4-7"`. NOT a wrapped `anthropic(...)` call. The AI SDK auto-routes the plain string through the gateway. Verify current model availability via `curl -s https://ai-gateway.vercel.sh/v1/models | jq -r '.data[].id' | grep claude-opus`.

6. **The `web_search` tool still imports from `@ai-sdk/anthropic`** even when the model is routed through the gateway. `anthropic.tools.webSearch_20250305({ maxUses: 5 })` produces a serializable tool definition the gateway passes through.

7. **Tailwind v4 has no `tailwind.config.ts`.** Config lives in `globals.css` via `@import "tailwindcss"` + `@theme {}`. Plan listed `tailwind.config.ts` as a file to create — that was for v3 conventions. Don't recreate it.

8. **shadcn was initialized with `--base radix`.** Default `base` (Base UI) has type conflicts with AI Elements. `components.json` records `"style": "radix-nova"`. Don't switch back.

9. **`pdf-parse@2.x` is class-based.** `new PDFParse({ data: new Uint8Array(buf) })`, then `await parser.getText()`, then `await parser.destroy()`. The plan's example used the v1 default-function export.

10. **`tsx` loads scripts as CJS**, so `import.meta.dirname` is undefined. The PDF ingest script uses a dual path: `typeof __dirname !== "undefined" ? __dirname : path.dirname(fileURLToPath(import.meta.url))`.

11. **The ingest script's npm command needs `--env-file=.env.local`** so it picks up the OIDC token: `"ingest:taxonomy": "tsx --env-file=.env.local scripts/ingest-taxonomy-pdf.ts"`.

12. **Post-tool-use validation hook is strict.** It will reject any `ANTHROPIC_API_KEY` reference, `generateObject` import, hyphenated model versions, etc. Stay aligned with the gateway/v6/dots conventions or the hook will fight you. Even `(0-100)` in a comment gets flagged as a "model slug uses hyphens" false positive — use en-dashes (`0–100`) or rephrase.

## Known UX gaps (deferred for v1)

These came out of Task 11's code quality review and the user's live test. They're real but non-blocking. Address them in a "polish" pass after Task 21 or as part of an explicit v1.1.

1. **`SessionHeader` timer freezes during silence.** `elapsed` is computed at render, but renders only happen on store updates — and silence produces no store updates. Fix: 1-second `setInterval` ticker (either a local `useState` or a `tick()` action in the store).
2. **No visible feedback on mic permission denial or Deepgram WS errors.** `onError: (e) => console.error(e)` logs an empty event object. User has to open DevTools to see anything. Fix: wrap `start()` in try/catch, route `onError` to a toast / `alert`, reset `isRecording` on failure.
3. **Mic stop sequence not documented.** `recorder.stop()` then `stream.getTracks().forEach(t => t.stop())` — a queued `ondataavailable` event might fire after stop. Harmless today but worth a comment.
4. **Mid-session JWT expiration.** Deepgram token is 10-min TTL. Long sessions will fail. Plan notes this as v1.1 follow-up. Fix: refresh the token + re-open the WS, or extend TTL.
5. **No fetch timeout on `/api/deepgram/token`.** If the route hangs, the session never starts. 5s `AbortSignal.timeout()` would help.
6. **`tests/smoke.test.ts`** tests `1+1=2` — doesn't exercise the jsdom + jest-dom setup. Replace with a basic `render` + `getByTestId` check when convenient.
7. **`lib/server/deepgram.ts`** had a header comment saying "validates at import time" but actually validates per-call. Either change the behavior or update the comment.
8. **`updateClaim` accepts any `Partial<ClaimCard>` patch** with no `id`-overwrite guard. Low-risk since the orchestrator is the only caller.
9. **`tests/reputation.test.ts`** doesn't cover the social-platforms branch (`twitter.com`, `x.com`, `facebook.com`, etc. → "low"). Trivial to add.

Don't fix any of these in the middle of subsequent tasks unless asked. Batch them at v1.1.
