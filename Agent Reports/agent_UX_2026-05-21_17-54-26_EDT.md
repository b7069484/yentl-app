## Scope

Focused user-experience audit of the latest locally running Yentl app from the user's perspective, with special attention to the passive-viewer use case:

- A viewer is watching a YouTube video or other media.
- Yentl is open beside it.
- The viewer may not click anything.
- The system should clearly show what is happening, what is true/false/uncertain, what rhetoric is being detected, and whether the session is healthy, useful, and worth trusting.

This is not a code-quality audit, not a CI audit, and not a feature implementation pass.

## Executive Summary

Yentl is already visually distinctive, calm, and coherent. The brand has a recognizable shape, the source picker is elegant, the YouTube flow is polished, the Watch layout has a real media/transcript relationship, and the extension side panel is the closest thing in the app to the intended "companion dashboard."

The core UX gap is not basic polish. The gap is signal strength. From across the room, the app currently reads as a refined research/workspace UI, not yet as a companion instrument panel that lights up with obvious good/bad, true/false, safe/risky, calm/urgent signals while a video plays.

The biggest concrete visual bug found: the Audio file and Media URL process buttons render as dark rectangles with effectively invisible text after the user has entered/staged valid input. This is a direct usability failure in two ingest paths.

The highest-value product move is to add an always-visible "Now / Verdict / Rhetoric / Confidence" signal board to Watch and the extension panel. It should be readable without interaction and should use strong, consistent colors and motion only when meaningful.

## Priority Findings

### P0 - Audio and Media URL primary buttons are unreadable

Evidence:

- Audio file staged state: the "Process audio" button appears as a dark/black bar with text hidden.
- Media URL valid-ready state: the "Process" button also appears as a dark/black pill with barely visible/hidden text.
- Code evidence: components/session/ingest-panes/audio-ingest-pane.tsx and components/session/ingest-panes/media-url-ingest-pane.tsx both use `bg-ink text-bg`; `text-bg` is not backed by a visible Tailwind token in this app.

User impact:

- Users who have successfully staged a file or URL lose the one action they need next.
- It makes the app feel broken precisely at the conversion point.

Recommended fix:

- Replace `text-bg` with an established visible token, probably `text-paper` or `text-white`, depending on the design system intent.
- Add a visual test/screenshot check for staged Audio file and valid Media URL states.

### P1 - The Watch view does not yet deliver "passive viewer" signals strongly enough

Evidence:

- Watch view successfully places video, transcript, source title, evidence queue, and counts together.
- However, the largest visual object is the video. The analysis signals are small: top metric cards, a subtle transcript panel, small chips, and an evidence queue below the player.
- Player border color currently tracks speaker identity, not factual/rhetorical risk. That is useful, but not the main signal a passive viewer expects.

User impact:

- From afar, a user can see that Yentl is "doing something," but not whether the video is currently mostly true, questionable, misleading, rhetorically heated, or evidence-backed.
- The user has to read text to understand the analysis.

Recommended fix:

- Add a persistent signal board near the media:
  - Current verdict: TRUE / MIXED / MISLEADING / FALSE / NO BACKING.
  - Rhetoric heat: Low / Rising / High.
  - Evidence confidence: Cited / Incomplete / Waiting.
  - Live status: Listening / Transcribing / Checking / Done.
- Make the board use large color blocks, simple words, and one or two numbers.
- Let the video frame pulse or change border tone on new high-impact findings, but avoid constant motion.

### P1 - The extension panel is the strongest companion surface, but it hides the big picture behind tabs

Evidence:

- At 420px side-panel width, the extension panel looks coherent and usable.
- The top status pill, transcript/claims/markers segmented tabs, and highlighted transcript text work well.
- But only one of transcript, claims, or markers is visible at a time.
- A passive viewer on Transcript cannot see the claim status summary. A viewer on Claims cannot see the marker status summary.

User impact:

- The panel is close to the goal but still asks the user to inspect tabs.
- The "what is Yentl saying right now?" signal is split across surfaces.

Recommended fix:

- Keep the tabs, but add an always-on mini signal strip above them:
  - Claim risk.
  - Rhetoric heat.
  - Evidence state.
  - New finding pulse.
- The tab content can remain detailed; the strip should carry the passive-viewer message.

### P1 - Browser-tab waiting flow is conceptually right but visually ambiguous

Evidence:

- Browser-tab ingest pane explains the "same page / extension / review in sync" model well.
- The checklist and functional samples are useful.
- After clicking "Check in-page extension," the page remains in the same preflight layout with only a small status message update.

User impact:

- A user may not know whether Yentl has started waiting, whether they should switch tabs, or whether something failed.
- The best use case is "open beside any video," but the app still feels like a setup document at this stage.

Recommended fix:

- After "Check in-page extension," switch into a bold waiting state:
  - Large "Waiting for Chrome extension" state.
  - Big blue/green/red status.
  - Clear "Go to the video tab and click Yentl" instruction.
  - Optional animated side-panel preview.

### P1 - Ingest paths are unevenly polished

Evidence:

- YouTube ingest is the most mature: preview card, valid URL recognition, progress stages, and fallback ladder.
- Browser-tab ingest has excellent instructional content and functional samples.
- Audio file, Text transcript, and Media URL are much sparser and visually weaker.
- Audio/Media URL also have the unreadable primary-button issue.

User impact:

- Users perceive source quality differently depending on the ingest path.
- The less polished paths feel like utilities, not part of the same premium companion experience.

Recommended fix:

- Bring Audio, Text, and Media URL up to the YouTube pattern:
  - Right rail preview/status.
  - Clear progress steps.
  - Recovery options.
  - "What happens next" microcopy.
  - Strong active/ready state.

### P2 - Overview is useful, but not yet a true dashboard from a distance

Evidence:

- Overview has claims/markers/speaker/session metric tiles, topic strip, and recent activity.
- The metric bars are elegant and information-rich.
- The "recent activity" list is too small to be a strong signal from afar.

User impact:

- Overview works for a user sitting at the screen.
- It is not yet the best "glanceable companion" state.

Recommended fix:

- Add a large top summary:
  - "Current read: Mostly factual, rhetoric rising."
  - "14 claims checked. 2 need backing. 4 rhetoric markers."
  - One big color state.
- Move detailed recent activity below that.

### P2 - Transcript highlights are precise but too subtle

Evidence:

- Transcript view uses underlines/backgrounds to mark claims by verdict.
- Watch transcript uses current line highlight and inline annotations.
- Extension panel highlights are more visible than the main transcript.

User impact:

- Up close, the highlights are useful.
- From a distance, the transcript still reads mostly as text, not as a live analysis surface.

Recommended fix:

- Use stronger left rails or badges per line when a line contains a claim/marker.
- Consider "new finding" flash animation for 1-2 seconds when a line receives a verdict/marker.
- Keep reduced-motion support.

### P2 - Rhetoric taxonomy can be confusing in the extension transcript

Evidence:

- Extension panel marker tab labels the validation item as "Rhetoric - clear."
- The transcript annotation labels the same item as "Bias - premature certainty."

User impact:

- Users may not understand whether Yentl found rhetoric, bias, or fallacy.
- This weakens trust in the marker language.

Recommended fix:

- Preserve the marker type consistently in transcript annotations.
- If "bias" is being used as a broader bucket, label it visibly as such and explain only in detail views.

### P2 - Session state is powerful but memory-only unless saved/restored

Evidence:

- Sample/live session state works inside the SPA when using header links.
- Hard navigations to /session?view=overview, /session?view=claims, etc. without a saved/restore id drop back to source selection because the in-memory session is gone.

User impact:

- A user who refreshes, shares, or opens a view directly may lose the active analysis context.
- This is especially risky for a tool whose value is the trail of evidence.

Recommended fix:

- Make Save more prominent once useful evidence exists.
- Consider automatic local draft persistence for active sessions.
- If a view URL has no live session, explain "This view needs an active or restored session" instead of returning silently to the picker.

### P2 - The legal/consent banner is correct but visually dominant on first-run mobile

Evidence:

- The consent banner is helpful and ethically important.
- On mobile source picker it consumes prime top space and visually competes with the product task.

User impact:

- The first mobile experience starts with caution rather than momentum.
- It may feel more administrative than fun/rewarding.

Recommended fix:

- Keep the warning, but consider a smaller inline disclosure after first load or a friendlier collapsed treatment.
- Keep dismiss behavior obvious and accessible.

### P2 - Internal project dashboards are polished but could overwhelm non-builder users

Evidence:

- /project/validation and /project/flows are well designed and very useful for development.
- They expose corpus, fixtures, flow atlas, and internal validation concepts.

User impact:

- Great for builders and reviewers.
- Too much for a normal Yentl user if discoverable as part of the public product.

Recommended fix:

- Keep them as internal/dev surfaces or clearly label them as validation tools.
- Do not mix them into the main passive-viewer journey.

## Recommended Product Direction

The next UX iteration should not be "more cards." The app already has cards. The next iteration should be a signal system.

Suggested companion signal model:

```text
LISTENING
Transcript arriving

CURRENT READ
Mixed / Needs evidence

FACTS
14 checked
2 need backing
0 false

RHETORIC
Rising
4 markers

LATEST
Speaker 2 - 01:41 - TRUE - politics
```

Design principles:

- Use simple state words: TRUE, FALSE, MIXED, NO BACKING, RHETORIC RISING, LISTENING, CHECKING.
- Make the state readable at 10 feet.
- Keep detailed explanations one click/scroll away.
- Use motion only for changes: new claim, new marker, error, or recovery.
- Distinguish factual verdict from rhetoric heat from source confidence.
- Do not make "red" mean everything. Red should be reserved for false/misleading, failure, or severe rhetoric. Yellow/amber should mean caution or incomplete evidence.

## Bottom Line

Yentl is close to having the right emotional tone: careful, elegant, non-threatening, and smart. The missing layer is a bold live signal language. For the user's stated goal, the product should feel less like "a beautiful transcript workspace" and more like "a calm instrument panel beside the media." The extension panel is the closest current surface; Watch and Overview should inherit that direction, with bigger, brighter, simpler live signals.
